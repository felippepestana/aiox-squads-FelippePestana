# Formatting Checklist — Peças Processuais

> **Squad:** legal | **Agent:** processual-writer
> **Uso:** Validação final antes de entregar/protocolar uma peça processual

---

## Checklist de Formatação e Qualidade

Execute cada item e marque: ✅ PASS | ❌ FAIL | ⚠️ VERIFICAR | N/A

---

### 1. Cabeçalho e Identificação

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 1.1 | Endereçamento | Em **NEGRITO + CAIXA ALTA**, alinhamento justificado, sem ponto final | | |
| 1.2 | Informações do processo | Bloco com Processo nº, Classe, partes — alinhado à esquerda | | |
| 1.3 | Rótulos do processo | Campos como "**Processo nº:**", "**Classe:**" em negrito | | |

---

### 2. Qualificação das Partes

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 2.1 | Nome da parte | Em **NEGRITO + CAIXA ALTA** | | |
| 2.2 | Texto de qualificação | Justificado, com dados civis completos | | |
| 2.3 | Representação processual | Advogado + OAB mencionados | | |
| 2.4 | Fundamento legal | Artigo do CPC/CLT citado em negrito para a peça apresentada | | |

---

### 3. Título da Peça

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 3.1 | Posicionamento | Após a qualificação, em linha exclusiva | | |
| 3.2 | Estilo | **NEGRITO + CAIXA ALTA** | | |
| 3.3 | Espaçamento | Linha em branco antes e depois | | |

---

### 4. Hierarquia de Seções

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 4.1 | Seções nível 1 | Algarismos romanos + **NEGRITO + CAIXA ALTA** + centralizado | | |
| 4.2 | Subseções nível 2 | Formato **{ROMANO}.{LETRA}.** + negrito + Title Case + alinhado à esquerda | | |
| 4.3 | Consistência numérica | Numeração romana sequencial (I, II, III, IV...) sem saltos | | |
| 4.4 | Consistência de subseções | Letras sequenciais (A, B, C...) dentro de cada seção | | |

---

### 5. Destaques e Ênfases

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 5.1 | Artigos de lei | Número do artigo E nome da lei em **negrito** | | |
| 5.2 | Valores monetários | Valores em R$ em **negrito** | | |
| 5.3 | Prazos e datas críticas | Prazos numéricos e datas relevantes em **negrito** | | |
| 5.4 | Doenças e diagnósticos | Nomes de doenças em **NEGRITO + CAIXA ALTA** | | |
| 5.5 | Conclusões jurídicas | Frases de conclusão do argumento em **negrito** | | |
| 5.6 | Nomes em listas | Nomes de terceiros/suscitados em listas numeradas em **NEGRITO + CAIXA ALTA** | | |
| 5.7 | Uso de negrito equilibrado | Negrito usado criteriosamente (não em todo o texto) | | |

---

### 6. Citações de Jurisprudência

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 6.1 | Formato de bloco | Citação em bloco recuado (`>`) | | |
| 6.2 | Itálico no trecho | Ementa/trecho da decisão em *itálico* | | |
| 6.3 | Aspas | Trecho entre aspas dentro do itálico | | |
| 6.4 | Referência completa | Tribunal, órgão, número, relator, data de julgamento e publicação | | |
| 6.5 | Autenticidade | Nenhuma referência fabricada ou presumida | | |
| 6.6 | Marcadores de verificação | Referências incompletas marcadas como [VERIFICAR ANTES DE PROTOCOLAR] | | |

---

### 7. Linguagem e Redação

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 7.1 | Ausência de "vem mui respeitosamente" | Substituído por "vem, respeitosamente," | | |
| 7.2 | Ausência de "DD." / "MM." | Não consta na peça | | |
| 7.3 | Ausência de arcaísmos listados | Nenhum arcaísmo da lista de anti-patterns | | |
| 7.4 | Carga argumentativa | Cada parágrafo serve a um propósito argumentativo | | |
| 7.5 | Conectivos lógicos | Uso adequado de "portanto", "com efeito", "nesse sentido", etc. | | |
| 7.6 | Ausência de redundâncias | Sem frases redundantes ou de preenchimento | | |
| 7.7 | Fatos com relevância processual | Apenas fatos que se conectam a fundamentos ou pedidos | | |

---

### 8. Pedidos e Fecho

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 8.1 | Pedidos em lista | Numerados (a), b)... ou 1., 2.,...) | | |
| 8.2 | Especificidade dos pedidos | Cada pedido identifica a providência concreta solicitada | | |
| 8.3 | Sem pedido genérico puro | Não consta apenas "tudo o mais que de direito" sem especificação | | |
| 8.4 | Fórmula de encerramento | "Termos em que, pede deferimento." ou equivalente adequado | | |
| 8.5 | Local e data | Cidade e data por extenso presentes | | |
| 8.6 | Assinatura | Nome do advogado e OAB identificados | | |

---

### 9. Alinhamento Geral

| # | Item | Critério | Status | Observação |
|---|------|----------|--------|------------|
| 9.1 | Corpo do texto | Todo o texto corrido em alinhamento justificado | | |
| 9.2 | Seções nível 1 | Títulos de seção principal centralizados | | |
| 9.3 | Subseções | Subseções alinhadas à esquerda | | |
| 9.4 | Espaçamento entre seções | Linha em branco entre seções principais | | |

---

## Alertas Específicos

> ⚠️ **SEMPRE VERIFICAR antes de protocolar:**
> - Números de processos e julgados citados
> - Datas de julgamento dos precedentes
> - Nomes dos relatores dos acórdãos
> - Vigência atual dos dispositivos legais citados
> - CNPJ/CPF das partes qualificadas
> - Endereços de terceiros suscitados
